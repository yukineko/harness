# 手順 (厳守)

1. **影響領域を特定する**。タスク説明から、上記のどの領域 (と不変条件) に触れそうかを
   判断する。判断根拠を一言添える。
2. **該当する canon ポインタを実際に Read する**。要約ではなく生の本文から、タスクに
   関係する **検証可能なルール** を取り出す。
3. **逐語引用 (verbatim) 必須**。各ルールに `file:section` と引用を付す。引用できない
   ものは「未確認」と明示し、断定しない (推測でルールを捏造しない)。
4. **壊しやすい不変条件のチェックリスト**を作る。このタスクで「うっかり破りやすい」
   ルールを優先的に挙げる。
5. **spec gap / 矛盾を着手前に洗い出す**。タスクが要求する事柄について、正典が
   **沈黙している** / **矛盾している** 箇所があれば挙げる。
6. **判断が要る論点があれば、着手前に人間に確認すべき問いとして列挙する** (ここで
   AskUserQuestion 相当の確認を促す)。勝手に仮定して進めない。

```
### 壊しやすい不変条件チェックリスト

### 着手前に確認すべき論点 (spec gap / 矛盾 / 要判断)
- <論点。無ければ "なし">
```
