# 網羅性批評の規律 (厳守)

1. **canon を実際に開き、検証可能なルール (反証可能な pass/fail を持つ規定) を棚卸し** する。
2. 各ルールについて、対応する実装が **存在し、かつ canon どおりか** を `Grep`/`Glob` +
   部分 `Read` で素早く確認する。**巨大ファイルの全文 Read は禁止**。
3. **逐語引用必須**。canon 本文から引用できないルールは列挙しない (推測で未照合を捏造しない)。
4. 出すのは **「canon にあるのに実装側で照合できなかった/欠落しているルール」** だけ。
   実装と整合していることを確認できたルールは出さない。主観的な良し悪しも出さない。
5. これは「もっと見るべきだった箇所」の指摘であり、確定した矛盾ではない。**判別がつかない
   ものは「未照合 (要確認)」として人間に上げる** (安全側)。

# 出力フォーマット (stdout)

以下の Markdown だけを出力する。前置き・後置きの会話文は不要。

```
## 網羅性批評: 未照合ルール

| canon のルール | 参照(file:節) | 逐語引用 | 状態(未照合/欠落の疑い) | needs_user |
|---|---|---|---|---|

## 見落とし候補 (人間の確認が要る)
- <未照合/欠落のルールを 1 行で。無ければ "なし">
```

# 機械可読マーカー (必須・厳守)

レポート本文の **最後** に、次の 3 行を **この厳密な形式** で出力する。

```
{{MARKER}}
needs_user: <yes|no>
summary: <見落とし候補の 1 行要約。改行なし。無ければ "なし">
```

- `needs_user` は **未照合/欠落の疑いが 1 件以上** あれば `yes`、無ければ `no`。
- canon が読めない・棚卸し不能のときは `needs_user: no` とし、summary に
  「批評不能 (理由)」と書く (誤って見落とし候補をでっち上げない)。
